【族語射手 03】手機版排行榜卡住後，我才看懂桌機規則不能直接搬過來

摘要 手機版明明可以玩，分數卻差點被排行榜擋在門外。 我原本以為是 API 壞掉，後來才發現，是桌機版的解鎖規則被硬套到手機無限模式上。

這張是真實手機版畫面。手機版是直式點字拼字，不是桌機版縮小。

我停下來的瞬間，是看到 cleared: true

手機版要接排行榜時，我看到一個提議：

cleared: true

天啊，這個欄位看起來很小。 小到你可能會想說，能送分就好。

可是我看到那一刻停了一下。

因為 cleared 在桌機版不是裝飾。 它代表玩家真的通過某個關卡，後端才會允許下一關或更高等級的分數被記錄。

手機版不是這樣玩的。

手機版是直式無限玩法。 玩家一局一局打，沒有桌機那種「第 1 關過了才解第 2 關」的序列解鎖。

如果我為了讓手機分數上榜，就塞一個假的 cleared: true，排行榜看起來會動。 但資料會開始說謊。

這就是我覺得不能過的地方。

問題不是分數送不出去

我一開始以為，問題是手機版送分 API 沒接好。

後來拆開看，真正的問題是： 桌機和手機的 level 不是同一件事。

桌機版的 `level` 它比較像關卡進度，和解鎖有關。高關卡必須先通過前面關卡。

手機版的 `level` 它比較像無限模式裡的里程碑或難度標記，不應該被桌機解鎖規則擋住。

這裡很容易踩坑。

欄位名字一樣，不代表意思一樣。 資料長得像，不代表規則可以共用。

AI 第一個想到的捷徑，是讓手機也送一個看起來像過關的狀態。 但那其實是在騙後端。

我真正要做的不是「讓它過」。 是讓後端知道：這筆分數來自手機，所以不要拿桌機的解鎖邏輯審它。

我這次的修法

最後修法很簡單，但判斷不簡單。

手機版送出自己的真實狀態。 後端看 platform。

如果是桌機，就保留原本的解鎖檢查。 如果是手機，就跳過不適用的桌機進度檢查。

大概像這樣：

分數送進來
→ 看 platform
→ desktop：檢查玩家是否已解鎖該關
→ mobile：不套桌機解鎖，改用手機排行榜規則

這件事提醒我，跨裝置不是只做 RWD。

有時候裝置不同，玩法也不同。 玩法不同，資料規則就不能假裝一樣。

驗證不能只看成功

這次我特別在意兩個測試。

第一個是手機成功案例。 新帳號在手機版送出高里程碑分數，應該可以成功。

第二個是桌機保護案例。 同一個新帳號如果在桌機版直接送高關卡分數，仍然要被拒絕。

我看兩種情況。

手機送出分數要成功。 手機無限模式不該被桌機規則誤擋。

桌機未解鎖高關卡要拒絕。 不能為了修手機，把原本的防作弊規則拆掉。

這兩個測試一起看，才知道修正有沒有真的對。

如果只有手機成功，我會擔心自己把門拆太大。 如果只有桌機安全，我又沒有解決手機玩家的問題。

所以我這次帶走的做法是： 修跨裝置功能時，一定要同時保留舊規則的反向測試。

新手可以怎麼做

如果你也要把同一款遊戲接到不同裝置，我會建議先畫一張表。

不用很工程。

就寫：

玩家怎麼操作？ 桌機通常是鍵盤和滑鼠，手機則是手指點選。

一局怎麼結束？ 桌機可能是關卡完成，手機可能是無限模式或生命歸零。

分數代表什麼？ 桌機常常代表關卡成績，手機可能更像單局表現。

是否需要解鎖？ 桌機需要，手機不一定。

排行榜要不要混在一起？ 這件事不能用感覺，要回到規則本身判斷。

只要這張表有一格不一樣，就不要急著共用同一條後端規則。

程式可以共用。 資料表也可能共用。 可是規則要先分清楚。

這次真正的代價

這次花掉的不是很多程式碼。

真正花時間的是承認： 手機版不是桌機版的附屬品。

它有自己的操作方式。 有自己的遊戲節奏。 也有自己的排行榜語意。

我不能因為欄位剛好叫 level，就把它塞回桌機那套世界觀。 也不能因為 AI 給了一個能通過的寫法，就把資料真相改掉。

做完這次，我更相信一件事： AI 可以很快幫我補程式，但欄位到底代表什麼，還是要人停下來判斷。
